Run this command to check git history for any accidentally committed secrets:
git log --all --full-history -- "**/.env" "**/.env.local"
If any results appear, the secrets in those commits must be rotated immediately (they are permanently in git history even if deleted). Also verify .env and .env.local are listed in .gitignore.

Every route that reads or mutates data must verify the session and role. Priority routes to audit:
• /api/therapist/* — must confirm role === 'therapist'
• /api/circle/schedule POST/DELETE — confirm therapist owns the entry
• /api/therapist/assign — confirm child belongs to a linked family
• /api/children — confirm child belongs to the authenticated parent
Pattern: const session = await getServerSession(authOptions); if (!session?.user) return 401;

Add Rate Limiting to AI Endpoints
Without rate limiting, anyone can spam AI routes and drain your Anthropic credits or perform brute-force attacks.
Install: npm install @upstash/ratelimit @upstash/redis
Apply to: /api/parent/ai-insights, /api/parent/ai-plan, /api/coaching, /api/therapist/generate, /api/auth/signin
Recommended limit: 10 requests per minute per IP using sliding window algorithm.

High Priority — Do This Week

Significant vulnerabilities that could be exploited by a motivated attacker

5 tasks · ~5 hrs total
Validate Ownership on All API ID Parameters
Never trust an ID from the client without confirming the authenticated user owns it. An attacker can send any ID and access other users' data (Insecure Direct Object Reference — OWASP #1).
Pattern to apply to every route: query the database with both the ID and the authenticated user's ID as a where condition before operating on the record. If no record found, return 404 (not 403 — don't confirm the record exists).
2 hrseffort
To Do
Add HTTP Security Headers in next.config.js
Security headers prevent a large class of browser-based attacks (XSS, clickjacking, MIME sniffing).
Add to next.config.js → headers() function:
• X-Frame-Options: SAMEORIGIN — prevents clickjacking
• X-Content-Type-Options: nosniff — prevents MIME sniffing
• Referrer-Policy: origin-when-cross-origin
• Permissions-Policy: camera=(self), microphone=()
• Content-Security-Policy — restricts script/style/connect sources
30 mineffort
To Do
Harden NextAuth Session Configuration
The default NextAuth config has weaker-than-necessary session settings.
Update lib/auth.ts to add:
• Session maxAge: 7 * 24 * 60 * 60 (7 days, not indefinite)
• Cookie httpOnly: true — prevents JS access to session cookie
• Cookie secure: true in production — HTTPS only
• Cookie sameSite: 'lax' — CSRF protection
15 mineffort
To Do
Sanitise User Input in AI Prompts (Prompt Injection)
AI routes accept user-controlled text (child notes, therapist goals, activity descriptions) and embed them directly into prompts sent to Claude. An attacker can craft input that manipulates the AI's behaviour (prompt injection attack).
Mitigations: strip HTML tags, enforce hard character limits (max 500 chars per field), and filter known injection phrases before including any user text in a prompt.

To Do

Preview deployments (from pull requests) should never have access to the production database or live API keys.
In Vercel dashboard → Settings → Environment Variables:
• Set DATABASE_URL, ANTHROPIC_API_KEY, PUSHER_SECRET to Production only
• Create a separate Neon database branch for Preview and use that connection string for non-Production environments
• Mark all secrets as Sensitive (write-only in Vercel UI)

Important — Do This Month

Security hardening and monitoring that significantly reduces long-term risk

5 tasks · ~3 hrs total
Add CSRF Protection to Custom API Routes
NextAuth handles CSRF for auth routes but your custom mutating API routes are unprotected. A malicious website could trick a logged-in user's browser into making requests to your API.
Add an Origin header check to all POST/PATCH/DELETE routes: verify the origin matches your own domain before processing the request.
45 mineffort
To Do
Harden the Circle Time Join Code System
The 6-character join code is currently valid indefinitely. Three improvements:
• Expiry: join code should auto-expire 2 hours after session ends
• Attempt limiting: max 5 wrong code attempts per IP per hour — return generic error, do not confirm code existence
• Status check: reject join requests if session status is anything other than waiting or active
1 hreffort
To Do
Set Up Error Monitoring with Sentry
Without monitoring, you won't know when your app is throwing errors or being attacked until a user reports it.
Install: npx @sentry/wizard@latest -i nextjs (free tier available)
Sentry captures: runtime exceptions, API errors, performance issues, and can alert you by email or Slack. Never log passwords, tokens, or children's personal data to Sentry — review the DSN configuration.
30 mineffort
To Do
Run Dependency Vulnerability Audit & Enable Dependabot
Outdated npm packages are a common attack vector. Known vulnerabilities are published publicly.
Run now: cd nextjs_space && npm audit — review and fix any high/critical findings
Auto-fix safe updates: npm audit fix
Enable Dependabot: GitHub repo → Settings → Security → Dependabot alerts → Enable. This automatically creates PRs when packages have known vulnerabilities.
20 mineffort
To Do
Add Security Event Logging
Log security-relevant events so you can detect patterns of abuse or breach attempts. Events to log: failed authentication attempts, repeated 401/403 responses, unexpected role escalation attempts, unusual API usage patterns.
Important: never log passwords, session tokens, join codes, or any PII (children's names, ages, health data) in logs — this itself would be a COPPA/HIPAA violation.
